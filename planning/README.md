# 기획 (Planning)

이 폴더는 악보생성기 서비스의 **"무엇을 만들 것인가"** 상세를 담습니다.

---

## 1. 기능 명세

### 1-1. 악보 자동 생성

| 항목 | 내용 |
|------|------|
| 입력값 | 곡명, 악기(드럼/기타/우쿠렐레/피아노), 연주 수준(입문/초급/중급) |
| 출력값 | 해당 악기·수준에 맞게 편곡된 악보 (PDF + 화면 렌더링) |
| 동작 | 곡명 입력 → 악기·수준 선택 → "악보 생성" 버튼 클릭 → 로딩(최대 10초) → 악보 표시 |
| 예외 처리 | 곡을 찾을 수 없는 경우 "유사 곡 추천" 제시, 생성 실패 시 재시도 안내 |

**수준별 편곡 기준:**

| 수준 | 드럼 | 기타 | 우쿠렐레 | 피아노 |
|------|------|------|---------|--------|
| 입문 | 4박자 기본 비트, 심벌 없음 | 오픈 코드만 | 4코드 이하 | 단선율 오른손만 |
| 초급 | 필 인 추가 | 바레코드 일부 포함 | 코드 전환 포함 | 양손, 단순 반주형 |
| 중급 | 다양한 리듬 패턴 | 솔로 라인 포함 | 핑거피킹 포함 | 원곡 편곡에 근접 |

---

### 1-2. 악보 재생 (미리듣기)

| 항목 | 내용 |
|------|------|
| 기능 | 생성된 악보를 MIDI 기반으로 재생 |
| 컨트롤 | 재생 / 일시정지 / 정지 / 구간 반복 |
| 시각 피드백 | 재생 중 현재 마디 하이라이트 |
| 템포 | BPM 설정값 반영 |

---

### 1-3. 메트로놈 BPM 설정

| 항목 | 내용 |
|------|------|
| 기능 | 악보 재생 속도(BPM) 조절 |
| 범위 | 40 ~ 200 BPM |
| 조작 | 슬라이더 + 직접 입력 |
| 기본값 | 원곡 권장 BPM 자동 설정, 사용자가 변경 가능 |
| 메트로놈 음 | 재생 시 박자 클릭음 On/Off 토글 |

---

### 1-4. 악보 저장 / 출력 (MVP 포함 여부: 포함)

| 항목 | 내용 |
|------|------|
| 저장 형식 | PDF 다운로드 |
| 출력 | 브라우저 인쇄 연동 |
| 제한 | MVP에서는 1일 3회 무료 저장 (이후 구독 전환 시 무제한) |

## 2. 사용자 시나리오

### 시나리오 A — "처음 기타를 배우는 대학생"

| 항목 | 내용 |
|------|------|
| **Who** | 22세 대학생, 기타 구매 후 독학 시작 2주 차 |
| **When** | 저녁 자유 시간, 유튜브 독학 중 |
| **Where** | 자취방, 노트북 사용 |
| **Why** | 좋아하는 노래를 직접 치고 싶은데 악보를 살 돈이 아깝고, 검색해도 수준에 맞는 악보가 없음 |
| **How** | 1. 서비스 접속 → 2. 곡명 "아이유 - 밤편지" 입력 → 3. 기타 / 입문 선택 → 4. 악보 생성 → 5. BPM 낮춰서 미리듣기 → 6. PDF 저장 후 연습 |
| **기대 결과** | 오픈 코드만으로 구성된 단순화 악보 제공, 연습 성공 경험 |

---

### 시나리오 B — "취미로 피아노를 다시 시작한 직장인"

| 항목 | 내용 |
|------|------|
| **Who** | 35세 직장인, 어린 시절 피아노 레슨 경험 있음 (10년 공백) |
| **When** | 주말 오전 |
| **Where** | 집, 디지털 피아노 앞 |
| **Why** | 공백 후 재시작, 원하는 팝송을 치고 싶으나 원곡 악보는 너무 어려움 |
| **How** | 1. 곡명 검색 → 2. 피아노 / 초급 선택 → 3. 악보 확인 → 4. BPM 60으로 낮춤 → 5. 구간 반복 기능으로 어려운 마디 집중 연습 |
| **기대 결과** | 양손 단순 반주형 악보로 연습 부담 감소 |

---

### 시나리오 C — "드럼 동아리 신입"

| 항목 | 내용 |
|------|------|
| **Who** | 19세 대학 드럼 동아리 신입 |
| **When** | 동아리 연습 전날 밤 |
| **Where** | 기숙사, 스마트폰 |
| **Why** | 동아리 과제곡 드럼 악보를 구해야 하는데 유료 사이트 가입하기 귀찮음 |
| **How** | 1. 모바일 접속 → 2. 과제곡 입력 → 3. 드럼 / 입문 선택 → 4. 악보 생성 및 미리듣기로 확인 → 5. PDF 저장 |
| **기대 결과** | 기본 비트 중심 단순 드럼 악보 즉시 획득 |

## 3. MVP 범위

### ✅ MVP에 포함 (필수)

| 기능 | 이유 |
|------|------|
| 악보 자동 생성 (4악기, 3수준) | 핵심 가치 명제 |
| 악보 화면 렌더링 | 생성 결과 확인 필수 |
| 악보 재생 (기본 재생/일시정지) | 음정 확인 없이는 활용 어려움 |
| BPM 설정 슬라이더 | 입문자 연습에 필수적 |
| PDF 저장 (1일 3회 무료) | 오프라인 연습 지원, 구독 전환 유도 |

### ❌ MVP 이후 (추후 개발)

| 기능 | 이유 |
|------|------|
| 악보 편집 (음표 직접 수정) | 구현 복잡도 높음, MVP 핵심 아님 |
| 구간 반복 재생 | 편의 기능, MVP 이후 추가 |
| 악보 공유 / 커뮤니티 | 소셜 기능은 트래픽 확보 후 |
| 악기 추가 (바이올린, 플루트 등) | 4악기로 충분히 검증 후 확장 |
| 멤버십 / 구독 결제 | 무료 MVP로 사용자 확보 먼저 |
| 모바일 앱 (iOS/Android) | 웹 MVP 검증 후 앱 전환 |
| 악보 즐겨찾기 / 히스토리 | 로그인 시스템 구축 후 |

## 4. 우선순위

개발 순서는 **사용자 핵심 흐름** 기준으로 설정합니다.

| 순위 | 기능 | 근거 |
|------|------|------|
| P0 | 악보 자동 생성 엔진 | 서비스의 존재 이유 |
| P0 | 악보 화면 렌더링 | 생성 결과를 눈으로 확인해야 신뢰 형성 |
| P1 | BPM 설정 | 없으면 입문자 사용 불가 수준 |
| P1 | 악보 재생 (미리듣기) | 악보 정확도 검증 수단, 이탈 방지 |
| P2 | PDF 저장 | 오프라인 활용 + 구독 전환 유도 트리거 |
| P3 | 구간 반복 재생 | 편의 기능, 우선순위 낮음 |

## 5. 비기능 요구사항 (MVP 기준)

| 항목 | 기준 |
|------|------|
| 악보 생성 응답 시간 | 10초 이내 |
| 지원 브라우저 | Chrome, Safari 최신 2버전 |
| 모바일 반응형 | 악보 조회·저장은 모바일에서도 가능해야 함 |
| 악보 정확도 | 내부 테스트 기준 사용자 만족도 80% 이상 |

## 6. 핵심 설계 결정 (Brainstorming 세션 확정)

| 항목 | 결정 |
|------|------|
| 악보 생성 방식 | 하이브리드 (DB 원곡 + AI 수준별 편곡) |
| 곡 검색 방식 | 텍스트 검색 + 자동완성 드롭다운 |
| 사용자 계정 | MVP 불필요 — 로그인 없이 바로 사용 |
| 렌더링 형식 | 악기별 최적 형식 자동 선택 |
| 공유 기능 | MVP 불필요 — PDF 저장만 제공 |
| 페이지 구조 | 심플 원페이지 (페이지 이동 없음) |
| 편곡 실패 처리 | 3회 이상 실패 시 다른 곡 검색 유도 |
